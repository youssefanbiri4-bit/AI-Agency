# AgentFlow-AI — Code Review Policy

**سياسة:** مراجعة الكود  
**تاريخ التفعيل:** 2026-07-11  
**المراجعة الدورية:** كل 3 أشهر

---

## 1. الهدف

ضمان جودة الكود من خلال مراجعة منهجية لكل تغيير قبل دمجه في الفرع الرئيسي.

## 2. النطاق

تطبق هذه السياسة على جميع التغييرات في الكود:

- ✅ كود إنتاجي جديد
- ✅ تعديل كود موجود
- ✅ إضافة مكتبات
- ✅ تعديل config files
- ✅ تعديل database migrations
- ❌ التغييرات في الوثائق فقط (لا تحتاج مراجعة)
- ❌ التغييرات في i18n labels (لا تحتاج مراجعة إذا كانت ترجمة فقط)

## 3. إجراءات المراجعة

### 3.1 قبل رفع الـ PR
يجب على الوكيل المنفذ التأكد من:
- [ ] `npm run build` ينجح
- [ ] `npm test` ينجح
- [ ] `npm run lint` لا توجد أخطاء
- [ ] `npm run typecheck` لا توجد أخطاء
- [ ] الكود يتبع conventions المشروع

### 3.2 محتوى الـ PR
يجب أن يحتوي الـ PR على:
- وصف واضح للتغيير
- قائمة الملفات المتأثرة
- نتائج الاختبارات
- Screenshots (للتغييرات في UI)

### 3.3 عملية المراجعة
1. **Architect Agent** يراجع الكود خلال 4 ساعات
   - ✅ → يمر إلى QA
   - ❌ → يرد مع ملاحظات محددة
2. **QA Agent** يختبر الجودة خلال 4 ساعات
   - ✅ → يمر إلى Security
   - ❌ → يرد مع ملاحظات
3. **Security Agent** يراجع الأمان خلال 2 ساعة
   - ✅ → يمر إلى Performance
   - ❌ → يرد مع بلوك
4. **Performance Agent** يراجع الأداء خلال 2 ساعة
   - ✅ → يمر إلى Documentation
   - ❌ → يرد مع ملاحظات
5. **Documentation Agent** يتحقق من التوثيق خلال 1 ساعة
   - ✅ → يمر إلى CTO
   - ❌ → يرد مع ملاحظات
6. **CTO Agent** يعطي الموافقة النهائية خلال 2 ساعة
   - ✅ → جاهز للنشر
   - ❌ → إعادة تقييم

### 3.4 مدة المراجعة
| نوع التغيير | المهلة | المبرر |
|-------------|--------|--------|
| تغيير صغير (< 20 سطر) | 2 ساعة | تغيير بسيط |
| تغيير متوسط (20-100 سطر) | 8 ساعات | مراجعة متوسطة |
| تغيير كبير (100+ سطر) | 24 ساعة | مراجعة شاملة |
| طارئ (hotfix) | فوري | لا مراجعة كاملة |

## 4. معايير الموافقة

### ✅ الموافقة
- الكود نظيف ومنظم
- TypeScript types صحيحة
- error handling مناسب
- لا duplicated code
- اختبارات كافية
- توثيق مناسب

### ❌ طلب تغييرات
- أخطاء في TypeScript أو logic
- اختبارات مفقودة
- conventions مخالفة
- code smell أو duplicated code
- نقص في error handling

### 🔴 بلوك
- ثغرات أمنية
- تغيير في API contracts دون تنسيق
- كود لا يمكن صيانته
- انتهاك خطير لـ project structure

## 5. استثناءات

### يمكن تخطي المراجعة لـ:
- Fix typo في التعليقات
- تغيير label في i18n
- إصلاح CSS بسيط (لا يغير layout)
- تحديث وثائق فقط

### لا يمكن تخطي المراجعة أبداً لـ:
- أي تغيير في business logic
- أي تغيير في database
- أي تغيير أمني
- أي تغيير في API contracts
